###############################################################################
# Правила отбора HTML-элементов при обходе HTML-документа
# Правила сгруппированы по доменам. Начало группы прав ил для домена начинается
# со строки:
# domain:<хост домена>
# после этого идет список правил фильтрации и классификации элементов.
#
# Правила фильтрации:
#
# <правило фильтра>     ::= +|-:<правило>
# <правило>             ::= <тип правила>:<спецификация правила>
# <тип правила>         ::= tag|class|attr
# <спецификация правила>::= <строка>
# Правила фильтрации начинаются с префикса "-" (не учитывать элемент при обходе)
# или "+" (учитывать элемент, даже если он попал под исключение другим правилом).
# Затем, после двоеточия идет тип правила и, после второго двоеточия,
# спецификация правила.
#
# tag - правило устанавливает соответствие с элементом по названию эго тега.
# В спецификации указывается имя тега для сопоставления.
#
# class - правило устанавливает соответствие с элементом по значению css-класса .
# В спецификации указывается выражение для сопоставления с каждым классом в атрибуте
# class. Звездочки в выражении могут задавать критерии сопоставления:
# * в начале - класс должен заканчиваться на заданную строку.
# * в конце - класс должен начинаться с заданной строки.
# * в начале и конце - класс должен содержать заданную подстроку.
# отсутствие * - класс должен быть равен заданной строке.
# Перед сравнением класс и выражение приводятся к нижнему регистру.
#
# attr - правило устанавливает соответствие с элементом по значению заданного атрибута.
# Пример: -attr:href:*prime.rambler.ru*
# В спецификации указывается имя атрибута и, после двоеточия, выражения для сопоставления.
# Звездочки в выражении могут задавать критерии сопоставления:
# * в начале - атрибут должен заканчиваться на заданную строку.
# * в конце - атрибут должен начинаться с заданной строки.
# * в начале и конце - атрибут должен содержать заданную подстроку.
# отсутствие * - атрибут должен быть равен заданной строке.
# Перед сравнением значение атрибута и выражение приводятся к нижнему регистру.
# При отсутствии атрибута у элемента сопоставление не производится.
#
# Классификаторы задают правила для выделения в тексте блоков соответствующих
# заголовкам и параграфам.
#
# <классификатор> ::= header|paragraph:<правило>
#
# header-классификатор задает правила, по которым определяется заголовок.
# paragraph-классификатор задает правила, по которым определяется параграф.
